    MCP Setup Guide

    This guide explains how to connect CodeGraph's MCP server to AI coding assistants so they can call CodeGraph tools directly from your IDE.

    How MCP Works (Brief Overview)

    MCP (Model Context Protocol) works like a plugin system for AI tools. CodeGraph runs as a local MCP server; your AI host (Claude Code, Cursor, Claude Desktop) connects to it and gains access to its three tools.

    AI Host (Claude Code)  ←──MCP stdio──→  CodeGraph MCP Server  ←──→  Neo4j

    The AI decides when to call the tools. You don't need to do anything — once connected, the AI will automatically use get_relevant_context when it thinks graph-based context would help.

    Prerequisites

    Before connecting any AI host:

    1. Neo4j is running with the GDS plugin installed
    2. The code graph has been built for your target repository (python -m scripts.demo_neo4j or scripts/rebuild_graph.py)
    3. python -m src.mcp_server starts without errors

    Test the server locally: 

    python -m src.mcp_server
# Should start and wait silently (no output = working correctly)
# Press Ctrl+C to stop

    Connecting to Claude Code (Claude CLI)

    Claude Code reads MCP server configuration from a JSON file.

    Step 1: Find or Create the Config File

    Claude Code looks for MCP config at: - macOS/Linux: ~/.config/claude/mcp_servers.json - Windows: %APPDATA%\claude\mcp_servers.json

    Create the file if it doesn't exist.

    Step 2: Add CodeGraph as an MCP Server

    {
  "mcpServers": {
    "codegraph": {
      "command": "python",
      "args": ["-m", "src.mcp_server"],
      "cwd": "/absolute/path/to/your/codegraph/directory"
    }
  }
}

    Important: cwd must be the absolute path to the CodeGraph project directory (where config.yaml lives). The server reads config.yaml from this directory.

    Step 3: Verify the Connection

    Start a Claude Code session in your target project directory and type:

    /tools

    You should see codegraph::get_relevant_context, codegraph::query_dependencies, and codegraph::get_graph_stats listed.

    Connecting to Cursor

    Step 1: Open MCP Settings

    In Cursor: Settings → MCP → Add MCP Server

    Step 2: Configure the Server

    Fill in: - Name: codegraph - Command: python -m src.mcp_server - Working Directory: /absolute/path/to/codegraph/

    Or edit ~/.cursor/mcp.json directly:

    {
  "mcpServers": {
    "codegraph": {
      "command": "python",
      "args": ["-m", "src.mcp_server"],
      "cwd": "/absolute/path/to/codegraph/"
    }
  }
}

    Step 3: Enable MCP in Cursor

    Go to Settings → search "MCP" → ensure Enable MCP is toggled on.

    Connecting to Claude Desktop

    Step 1: Edit the Config File

    macOS: ~/Library/Application Support/Claude/claude_desktop_config.json Windows: %APPDATA%\Claude\claude_desktop_config.json

    Step 2: Add the Server

    {
  "mcpServers": {
    "codegraph": {
      "command": "python",
      "args": ["-m", "src.mcp_server"],
      "cwd": "/absolute/path/to/codegraph/"
    }
  }
}

    Step 3: Restart Claude Desktop

    The MCP server is started fresh each time Claude Desktop launches. After restarting, you should see the CodeGraph tools in the tool menu (hammer icon).

    Using CodeGraph Tools in a Session

    Once connected, the AI uses tools automatically. You can also prompt it explicitly:

    Explicit prompts that trigger tool use: - "Use CodeGraph to find relevant context for this authentication bug" - "What are the direct callers of the authenticate function?" - "Get the graph stats for this codebase"

    Natural prompts the AI will auto-decide to use tools for: - "Fix the token expiry bug in the auth service" - "Refactor the payment processing flow" - "Why is the login failing?"

    Example session: 

    You: Fix the authentication timeout bug. Users are getting logged out after 5 minutes.

Claude: Let me get relevant context from the code graph first.

[calls get_relevant_context(
    task_description="authentication timeout bug users logged out 5 minutes",
    mentioned_entities=["authenticate", "timeout", "session"]
)]

I found 8 relevant functions. Looking at TokenManager.validate_token (score: 0.12),
I can see the session timeout is hardcoded to 300 seconds...

    Pointing CodeGraph at a Different Repository

    By default, CodeGraph indexes the project_root specified in config.yaml. To analyze a different codebase:

    1. Update config.yaml: 

      project_root: "/path/to/target/repository"

    2. Rebuild the graph: 

      python -m scripts.rebuild_graph

    3. The MCP server automatically uses the new graph for subsequent queries.

    Troubleshooting

    Tools don't appear in the AI host: - Check that cwd in the MCP config is the correct absolute path - Verify python -m src.mcp_server works in that directory - Some hosts require restarting after MCP config changes

    Tools appear but return errors: - Neo4j must be running when the AI calls the tools - Check that the graph is built: run get_graph_stats explicitly

    "Module not found" errors: - The Python environment used by the MCP config must have CodeGraph installed - Use the full Python path if needed: /path/to/venv/bin/python

    See Troubleshooting → for more.
