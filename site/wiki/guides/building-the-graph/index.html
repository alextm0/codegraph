    Building the Graph

    This guide explains how to index a Python repository into the CodeGraph Neo4j database. The graph must be built before any retrieval queries can be run.

    Overview

    Building the graph has three steps:

    1. Parse — tree-sitter reads all .py files and extracts entities
    2. Resolve — imports and calls are connected across files
    3. Load — nodes and edges are written to Neo4j (idempotently)

    The graph is built once per repository state. Subsequent MCP queries run in under 250ms. When the code changes, rebuild the graph.

    Option 1: Demo Script (Quickstart)

    The demo script indexes a small fixture project and prints a summary. Good for testing that everything works:

    python -m scripts.demo_neo4j

    This indexes tests/fixtures/simple_project/ — a hand-crafted project designed for testing.

    Option 2: Rebuild Script (Production)

    To index a real repository, use the rebuild script:

    python -m scripts.rebuild_graph

    This reads project_root from config.yaml, clears the existing graph, rebuilds from scratch, and creates the GDS projection.

    Before running: Set project_root in config.yaml to the repository you want to index:

    project_root: "/path/to/your/python/project"

    Option 3: Programmatic Build

    For fine-grained control or integration into your own scripts:

    import yaml
from src.parser.python_parser import create_parser, parse_directory
from src.graph.connection import create_driver, load_full_config
from src.graph.graph_builder import build_graph, clear_database
from src.graph.ppr import create_gds_client, project_graph

# Load config
config = load_full_config('config.yaml')

# Create connections
driver = create_driver(config['neo4j'])
gds = create_gds_client(config['neo4j'])

# Parse the codebase
parser = create_parser()
all_entities = parse_directory(config['project_root'], parser)

print(f"Parsed {len(all_entities)} files")

# Clear old graph and build new one
clear_database(driver)
build_graph(driver, all_entities)

print("Graph built. Creating GDS projection...")

# Create the in-memory projection for PPR
project_graph(gds)

print("Done.")
driver.close()

    Rebuild After Code Changes

    The graph is not automatically updated when code changes. You must rebuild manually.

    For a full rebuild: 

    python -m scripts.rebuild_graph

    For incremental updates (not yet implemented — see Future Work →), you would only rebuild changed files.

    When to rebuild: - After any significant refactoring - When adding new files to the project - Before running evaluations - When results feel stale or wrong

    Checking the Build

    After building, verify the graph is populated:

    # Via Neo4j Browser
MATCH (n) RETURN labels(n), count(*) ORDER BY count(*) DESC

# Via Python
python -c "
from src.graph.connection import create_driver, load_full_config
config = load_full_config('config.yaml')
driver = create_driver(config['neo4j'])
with driver.session() as session:
    result = session.run('MATCH (n) RETURN count(n) AS total')
    print(f'Total nodes: {result.single()[\"total\"]}')
driver.close()
"

    Configuring What Gets Indexed

    Exclude patterns control what gets skipped:

    # config.yaml
exclude_patterns:
  - "__pycache__"
  - ".venv"
  - ".git"
  - "*.pyc"
  - ".pytest_cache"
  - "tests"        # add this to skip test files
  - "docs"         # add this to skip documentation

    Pattern matching uses pathspec (gitignore-style syntax). You can use wildcards (*.pyc), directory names (__pycache__), and paths.

    Performance Reference

    Repository Files Nodes Edges Parse Time Load Time
    Small project (1k LOC) ~10 ~200 ~400 < 0.1s < 1s
    Medium project (10k LOC) ~100 ~2,000 ~8,000 ~1s ~5s
    Flask (65k LOC) ~200 ~5,000 ~20,000 ~2s ~10s
    Django (300k LOC) ~900 ~20,000 ~80,000 ~8s ~40s

    These are rough estimates. Actual times depend on hardware and graph complexity.
