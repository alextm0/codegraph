    Glossary

    Key terms used throughout the CodeGraph documentation and codebase.

    A

    Abstract Syntax Tree (AST) A tree representation of source code's syntactic structure. Each node represents a language construct (function definition, variable assignment, function call, etc.). tree-sitter produces ASTs from Python source files.

    Alpha (α) The restart probability in Personalized PageRank. With probability α, the random walker teleports back to the seed nodes instead of following an edge. Default: 0.15. Controls how tightly results cluster around seeds.

    Ablation study An experiment that removes or changes one component of a system to measure that component's contribution. Example: running PPR without CALLS edges to measure how much call relationships contribute to retrieval quality.

    B

    BM25 (Best Match 25) A text ranking algorithm from information retrieval. Given a query and a set of documents, BM25 ranks documents by how well they match the query — considering term frequency, document length, and inverse document frequency. Used in CodeGraph for keyword-based seed extraction.

    Base weight The starting edge weight before IDF adjustment. Base weights by edge type: INHERITS_FROM=1.0, CALLS=0.8, IMPORTS=0.5, CONTAINS=0.4.

    C

    Call resolution The process of connecting a function call site (e.g., hash_password(pw) inside authenticate()) to the actual function definition (e.g., hash_password in src/utils/crypto.py). One of the two resolution problems — the other is import resolution.

    Cypher Neo4j's query language for graph databases. Analogous to SQL but for graph queries. Pattern-matching syntax: MATCH (a)-[:CALLS]->(b) finds all pairs where a calls b.

    ContextResult The output type of the retrieval pipeline. Contains: entity_name, qualified_name, file_path, line_number, source_code, relevance_score.

    CONTAINS edge A graph edge representing structural containment: File → Class (the file defines this class), File → Function, or Class → Method.

    D

    Damping factor In PPR/PageRank: damping_factor = 1 − α. The probability of continuing the random walk at each step. Neo4j GDS uses damping factor instead of alpha. Default: 0.85.

    Dependency graph A graph where nodes are code entities and edges represent dependencies between them (calls, imports, inheritance). CodeGraph builds a dependency graph for Python repositories.

    E

    Edge A connection between two nodes in the graph. In CodeGraph: CONTAINS, IMPORTS, CALLS, and INHERITS_FROM. Edges have a weight property used by PPR.

    Entity A named code element extracted from source files. The four entity types in CodeGraph: File, Class, Function, Method.

    Entity extraction The process of walking an AST and pulling out named entities and their properties. Implemented in src/parser/extractors.py.

    F

    FileEntities The data model produced by parsing a single Python file. Contains lists of FunctionEntity, ClassEntity, MethodEntity, ImportEntity, and CallEntity objects.

    Fan-in The number of incoming edges to a node. High fan-in = many things depend on this entity. Used in IDF reweighting: high fan-in nodes get their incoming edges downweighted.

    G

    GDS (Graph Data Science) Neo4j's plugin library providing graph algorithms including Personalized PageRank. Requires a separate installation. The GDS Python client (graphdatascience) wraps the plugin's REST API.

    GDS Projection An in-memory copy of a subset of the Neo4j graph, optimized for algorithm execution. Created once per graph; PPR can run many times against the same projection. Named "codegraph" in this project.

    I

    IDF (Inverse Document Frequency) A text retrieval concept: terms appearing in many documents are less informative. Applied to graphs: edges to highly-connected nodes (high in-degree) are downweighted. Used to reduce utility function noise.

    Import resolution The process of turning a Python import string (e.g., "src.utils.crypto") into an actual file path on disk (src/utils/crypto.py). One of the two resolution problems.

    In-degree The number of incoming edges to a node. A function with in-degree 200 is called by 200 other functions — it's likely a utility function and gets IDF-downweighted.

    M

    MCP (Model Context Protocol) An open standard (Anthropic, November 2024) for communication between AI applications and external tools/data sources. CodeGraph implements an MCP server exposing three tools: get_relevant_context, query_dependencies, get_graph_stats.

    Method A function defined inside a class body. Stored separately from standalone Function entities. Qualified name format: file_path::ClassName.method_name.

    MRR (Mean Reciprocal Rank) An evaluation metric: the average of 1/rank of the first correct result across all queries. MRR = 1.0 means the correct answer always appears first. MRR = 0.5 means it appears at rank 2 on average.

    N

    Neo4j A graph database that stores data as nodes and edges. CodeGraph uses Neo4j as its persistent store for the code dependency graph. Neo4j Desktop provides a free development license.

    Node A code entity in the graph: File, Class, Function, or Method. Nodes have properties (name, file_path, line_number, etc.) and a unique qualified_name.

    P

    PageRank An algorithm (Google, 1998) that ranks nodes in a graph by simulating a random walker who follows edges and occasionally teleports to random nodes. The visit frequency converges to an importance score for each node.

    Personalization vector The weighted distribution of probability mass across seed nodes used to initialize PPR. Weights sum to 1.0. Nodes with higher weight get more starting probability mass and pull the walk more strongly.

    Personalized PageRank (PPR) A variant of PageRank where teleportation goes to specific seed nodes (not random nodes). Produces node rankings that reflect structural relevance to the seeds, not global importance.

    Post-processing The two steps applied to PPR results before returning them to the AI agent: (1) IDF edge reweighting to reduce utility function noise, and (2) token budgeting to fit results within a token limit.

    PPRConfig A configuration object for PPR parameters: damping_factor, max_iterations, tolerance, top_k.

    PPRResult A single result from PPR execution: node qualified name + relevance score.

    project_root The root directory of the codebase being indexed, specified in config.yaml. All file paths are relative to this directory.

    Q

    Qualified name A globally unique identifier for a code entity, combining the file path and entity name: - Function: src/utils/helpers.py::validate_email - Method: src/services/auth_service.py::AuthService.authenticate - Class: src/models/user.py::User

    R

    Recall@k An evaluation metric: the fraction of relevant results (gold files) that appear in the top-k retrieved results. Recall@10 = 0.70 means 70% of the files that needed changing appeared in the top 10 results.

    Resolution rate The percentage of imports (or calls) that the resolver successfully maps to a graph entity. Reported during graph building. Typical rates: 80–90% for imports, 60–80% for calls.

    S

    Seed nodes The starting nodes for PPR. Seeds are extracted from the task description, explicitly mentioned entities, and optionally the current file. Seeds with high weight pull the PPR walk toward their region of the graph.

    Seed selection The multi-signal process of identifying which graph nodes to use as PPR seeds. Three signals: entity name matching (weight 0.6), BM25 keyword search (weight 0.3), current file hint (weight 0.1).

    SWE-bench Lite A dataset of 300 real Python bug reports with associated repository states and gold patches (the code changes that fixed each bug). Used to evaluate CodeGraph's retrieval quality.

    T

    Token budget The maximum number of tokens of source code to include in the MCP response. Default: 6,000 tokens. PPR results are added in score order until the budget is consumed.

    tree-sitter A parser library that converts source code into an Abstract Syntax Tree. Used in CodeGraph to parse Python files. Supports 100+ languages, handles broken code gracefully.

    U

    UNDIRECTED projection In GDS, projecting edges as undirected means PPR can flow in both directions along each edge. Used in CodeGraph so that the random walk can find both upstream (callers) and downstream (callees) neighbors of the seed.

    W

    Weight (edge) A numeric property on graph edges representing coupling strength. Used by PPR during the random walk — higher weight means more probability mass flows along that edge. Base weights by type: INHERITS_FROM=1.0, CALLS=0.8, IMPORTS=0.5, CONTAINS=0.4, adjusted by IDF.
