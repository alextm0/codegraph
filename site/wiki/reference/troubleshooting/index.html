    Troubleshooting

    Common issues and how to fix them.

    Neo4j Connection Issues

    "Connection refused" / "ServiceUnavailable"

    Symptom: 

    neo4j.exceptions.ServiceUnavailable: Failed to establish connection to ('localhost', 7687)

    Cause: Neo4j isn't running.

    Fix: - Neo4j Desktop: Open the app and click Start on your database - Docker: docker start neo4j (or run the full docker command from Installation) - Verify it's running: curl http://localhost:7474 should return a JSON response

    "Authentication failed"

    Symptom: 

    neo4j.exceptions.AuthError: {code: Neo.ClientError.Security.Unauthorized}

    Fix: Check config.yaml — the password field must match what you set when creating the Neo4j database. If you forgot your password, reset it via Neo4j Desktop (database settings → Reset password).

    "Database does not exist"

    Symptom: 

    neo4j.exceptions.ClientError: Database 'mydb' does not exist.

    Fix: Set database: "neo4j" in config.yaml. The default Neo4j installation only has a database named neo4j. You only need to change this if you deliberately created a named database.

    GDS Issues

    "There is no procedure with the name gds.graph.project"

    Symptom: Running any GDS operation fails with a "procedure not found" error.

    Cause: The GDS plugin isn't installed.

    Fix: - Neo4j Desktop: Go to your database → Plugins tab → Install Graph Data Science Library → restart - Docker: Add -e NEO4J_PLUGINS='["graph-data-science"]' when running the container

    Verify fix: Run RETURN gds.version() in the Neo4j Browser — it should return a version string.

    "The graph 'codegraph' does not exist"

    Symptom: PPR fails because the GDS projection hasn't been created.

    Fix: Build the graph first (which creates the projection), or create it manually:

    CALL gds.graph.project(
  'codegraph',
  ['Function', 'Method', 'Class', 'File'],
  {
    CALLS:         {orientation: 'UNDIRECTED', properties: 'weight'},
    IMPORTS:       {orientation: 'UNDIRECTED', properties: 'weight'},
    INHERITS_FROM: {orientation: 'UNDIRECTED', properties: 'weight'},
    CONTAINS:      {orientation: 'UNDIRECTED', properties: 'weight'}
  }
)

    "Graph 'codegraph' already exists"

    Symptom: When rebuilding the graph, the projection creation fails because an old one exists.

    Fix: Drop the old projection first:

    CALL gds.graph.drop('codegraph', false)

    The false parameter means "don't fail if it doesn't exist." The pipeline's ensure_graph_ready() handles this automatically, but if you're building manually, run this first.

    Parsing Issues

    "SyntaxError: Could not parse file"

    Symptom: A file fails to parse.

    Behavior: tree-sitter is error-tolerant and will partially parse broken files. The error is logged as a warning and the file is skipped. Parsing continues for all other files.

    Fix: Check the specific file for syntax errors: 

    python -m py_compile path/to/file.py

    "0 entities extracted from X files"

    Symptom: Graph build completes but the database is empty.

    Causes: 1. project_root in config.yaml points to the wrong directory 2. All files are matched by exclude_patterns 3. The directory contains no .py files

    Fix: 

    # Verify which files would be parsed
python -c "
from src.parser.python_parser import parse_directory, create_parser
parser = create_parser()
results = parse_directory('.', parser)
print(f'Found {len(results)} files')
for r in results[:5]:
    print(r.file_path)
"

    Retrieval Issues

    "Empty results from get_relevant_context"

    Symptom: The MCP tool returns an empty list or very few results.

    Check 1: Is the graph built? Run in Neo4j Browser: 

    MATCH (n) RETURN count(n)
    If this returns 0, the graph hasn't been built.

    Check 2: Did seed extraction find anything? 

    from src.retrieval.seed_selection import extract_seeds
seeds = extract_seeds(
    task_description="your task",
    mentioned_entities=["some_function"],
    current_file=None,
    all_entities=[],  # pass your FileEntities here
    config=config['seed_selection']
)
print(seeds)

    Check 3: Is the GDS projection up to date? Drop and recreate it after rebuilding the graph.

    "PPR scores are all near 0"

    Symptom: PPR runs but returns scores like 0.0001 for everything.

    Cause: Seeds weren't matched to actual nodes in the graph. PPR with no valid seed nodes assigns tiny baseline scores to everything.

    Fix: Check that the entity names in mentioned_entities match names in the graph: 

    MATCH (n {name: "your_entity_name"})
RETURN n.name, n.file_path
    Entity names are case-sensitive.

    Test Failures

    Tests fail with "No module named 'src'"

    Fix: Install the package in editable mode from the project root: 

    python -m pip install -e ".[dev]"

    Neo4j tests are skipped

    Behavior: Tests marked @neo4j_required skip automatically when Neo4j isn't running. This is expected. Start Neo4j to run the full test suite.

    Check which tests skipped: 

    python -m pytest tests/ -v 2>&1 | grep SKIP

    Tests fail with "graphdatascience version mismatch"

    Fix: Upgrade the GDS Python client: 

    pip install --upgrade graphdatascience

    Make sure the GDS plugin version in Neo4j and the Python client version are compatible. GDS Python client 1.7+ works with GDS plugin 2.x.

    MCP Server Issues

    Claude Code doesn't see the "codegraph" tools

    Cause: The MCP server isn't configured in Claude Code's settings.

    Fix: See MCP Setup Guide → for the full configuration walkthrough.

    MCP server starts but tools return errors

    Check the server logs: When running python -m src.mcp_server, errors are printed to stderr. Look for: - ConnectionRefusedError → Neo4j isn't running - KeyError: 'neo4j' → config.yaml is missing or malformed - gds.graph.exists error → GDS not installed

    Getting More Help

    1. Check the logs: Set logging.basicConfig(level=logging.DEBUG) for verbose output
    2. Run tests: python -m pytest tests/test_graph_resolution.py -v (no Neo4j needed)
    3. Inspect the graph: Use the Neo4j Browser at http://localhost:7474
    4. Check decisions: DECISIONS.md explains why specific choices were made
